# Style Rotation and Novelty

Use this when daily experiments start to feel samey. Treat the entire experiments
gallery as shared creative history, regardless of which agent made each page.

## Signal

- Similar palette, framing, or interaction patterns keep recurring.
- The work is technically sound but not adventurous enough.
- Review feedback asks for more variation, risk, or surprise.

## Practice

1. Change the core metaphor, not just the labels.
2. Vary the interaction model: drag, reveal, simulate, compose, orbit, animate, or disrupt.
3. Swap the visual register: bureaucratic, biological, nocturnal, civic, ceremonial, playful, or industrial.
4. Avoid reusing the same HUD layout, card structure, or text hierarchy unless it is the point.
5. If the concept feels safe, add one constraint that forces a new shape.
6. Name the conceptual skeleton before implementation. Reject the idea if it merely
   reskins a recent loop such as ambient scene, central prop, one escalating action,
   changing counter, and accumulating actors.
7. Rotate the visitor's role as well as the setting: observer, performer, navigator,
   editor, conspirator, builder, translator, or saboteur.
8. Review every experiment page in the gallery before committing to a concept. Include
   Codex, Claude, Antigravity, and any future agents: originality is gallery-wide, not
   agent-local.
9. Reject a concept when another page already shares its premise, visitor role,
   interaction grammar, or dominant visual composition. A different news seed or copy
   treatment is not enough.

## Checks

- Ask whether the next page would still feel distinct if all copy were removed.
- Check that the motion language is not a clone of the previous experiment.
- Keep the joke legible, but let the form take a different route each time.
- Compare the proposed interaction loop against the full experiments gallery before coding.
- Record the nearest gallery neighbors and state why the new idea is structurally
  different before writing tests.
